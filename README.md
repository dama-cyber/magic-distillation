# fiction-distiller

> 稗官风骨全维蒸馏之器——自诸说部文字中提取格调之印，存其本味，别撰新篇。

## 这是什么

一个支持 **opencode** 和 **Claude Code** 双平台的小说风格蒸馏技能。核心能力很简单：**你给一篇小说，它拆出风格密码，再用这套密码写出全新的故事。**

不是抄袭情节，不是改写人物——只继承叙事节奏、句式习惯、修辞偏好、感官侧重这些风格层面的东西。输出附原创性声明，每一条都过自检。

## 架构

三层懒加载知识库，按需调用，不一次全塞进上下文：

```
🔴 基础层（始终加载）
├── general-style/      文体学 · 叙事学 · 修辞学 · 计量学 · 成像理论
└── author-fingerprint/ 词汇 · 句式 · 修辞 · 叙事声音 · 标点 · 跨作品指纹

🟡 体裁层（按文本类型加载）
├── long-fiction/       长篇：冲突模型 · 叙述层面 · 结构设计 · 文体 · 修改
├── short-fiction/      短篇：唯一效果 · 战略选择 · 细节真实 · 结构 · 美学
└── net-style/          网文：流派 · 爽点机制 · 世界观 · 人设 · 节奏 · 情感策略

🟢 方法层（创作阶段加载）
└── ai-distillation/    特征提取 · 指纹验证 · 模仿策略 · 原创性评估
```

共 32 个主题文件，每个约 3500 字，覆盖从亚里士多德到网文黄金三章的全部谱系。

## 懒加载协议

四步加载，每一步完成才触发下一步，禁止一次性全加载：

| 步骤 | 触发时机 | 加载范围 | 文件数 |
|------|---------|---------|-------|
| ① 索引 | 始终首先 | 仅 index.md 摘要 | 1 |
| ② 基础层 | 文本接收后 | general-style + author-fingerprint | 11 |
| ③ 体裁层 | 蒸馏开始时 | 按文本类型选一组 | 5-7 |
| ④ 方法层 | 创作阶段 | ai-distillation | 4 |

## 执行流程

```
步骤 1: 知识索引构建     → 按三层优先级读取 index.md，构建分析矩阵
步骤 2: 文本接收与切分   → 长篇逐章 / 短篇全文 / 多篇对比
步骤 3: 全维度风格蒸馏   → 11 个维度解构 → 输出风格模型卡
步骤 4: 原创故事构思     → 原创隔离原则，只继承风格规则
步骤 5: 原创模仿创作     → 以"我就是原作者在写新故事"的视角写作
步骤 6: 原创性自检       → 6 项红线检查 + 原创性声明
```

生成的内容按类型逐章保存到各平台 `output/` 下：`风格模型卡-《作品名》.md`、`第01章-《故事名》.md`……

## 风格模型卡

蒸馏的最终产物——每个卡包含：

- **核心风格标签**（5-8 个）
- **15 条必遵风格规则**（可量化、可自检）
- **风格指纹小结**（300 字凝练）
- **反面清单**（写作时绝对禁止的 5 个行为）

## 红线

```
❌ 跳过知识索引
❌ 复制原文任何连续句子（≥10字）
❌ 沿用原文情节/场景/对白/专名
❌ 长篇以整体印象代替逐章分析
❌ 虚构搜索结果
```

## 使用方式

在 opencode 或 Claude Code 的对话中提供小说文本即可。系统自动判断篇幅、选择知识路径、执行蒸馏到创作的完整 pipeline。

两个平台共享同一套位于 `.opencode/skills/fiction-distiller/knowledge/` 的知识库，通过各自 SKILL.md 中的 `@skill:knowledge/...` 引用懒加载。

## 项目结构

```
.opencode/skills/fiction-distiller/
├── SKILL.md           ← opencode 技能入口（YAML frontmatter + 完整指令）
├── knowledge/         ← 共享知识库（32 个主题文件，按三层优先级组织）
│   ├── index.md
│   ├── general-style/
│   ├── author-fingerprint/
│   ├── long-fiction/
│   ├── short-fiction/
│   ├── net-style/
│   └── ai-distillation/
└── output/            ← opencode 生成内容输出目录

.claude/skills/fiction-distiller/
├── SKILL.md           ← Claude Code 技能入口（通过 @ 导入引用同一套指令）
└── output/            ← Claude Code 生成内容输出目录

CLAUDE.md              ← Claude Code 项目级自动发现桥接文件
```
