# 06-short-pipeline：短篇七步工作流程

## 1. 总览

短篇换元仿写的完整流程分为七步：

```
输入原文短篇
    ↓
[Step 1] 字数/结构分析 → 字数预算表
    ↓
[Step 2] 五维骨架提取 → 骨架卡
    ↓
[Step 3] 关节点/变量标注 → 换元清单
    ↓
[Step 4] 新变量方案设计 → 新变量方案
    ↓
[Step 5] 按骨架逐段仿写 → 新短篇初稿
    ↓
[Step 6] 字数对齐修正 → 修正版
    ↓
[Step 7] 反检测 & 原创性验证 → 终稿 + 报告
```

## 2. Step 1：字数/结构分析

### 2.1 目标

分析原文的字数结构和段落分布，为后续生成设定字数预算。

### 2.2 输出

- 原文总字数
- 段落数及每段字数
- 每段功能标注
- 句子数及句长分布
- 对话/叙述/描写字数占比
- 字数预算表（见 `word-count-control/02-budget-table.md`）

### 2.3 操作方法

1. 使用 `word_count_analyzer.py` 自动分析
2. 人工复核每段功能标注
3. 确认预算表合理后进入下一步

## 3. Step 2：五维骨架提取

### 3.1 目标

提取原文的叙事骨架，形成可在新篇中借鉴的"写法卡"。

### 3.2 输出

骨架卡（YAML 格式），包含：
- 总字数、段落数
- 结构（段落功能、字数、情绪）
- 节奏（句长分布、段落长度分布、紧张舒缓比）
- 视角（人称、聚焦、范围）
- 语言（句式、修辞、对话/叙述/描写比例、标点习惯）
- 情绪曲线（段落级情绪强度与类型）

### 3.3 操作方法

1. 使用 `skeleton_extractor.py` 自动提取 + LLM 辅助
2. 人工检查骨架卡是否包含具体内容（应只包含写法）
3. 如发现具体内容残留，删除后重新生成

## 4. Step 3：关节点/变量标注

### 4.1 目标

识别原文中所有具体内容元素，并规划替换方案。

### 4.2 输出

换元清单（Markdown 表格），包含：
- 关节点编号
- 类型（人物/地点/事件/道具/台词/意象/关系）
- 原文内容（参考用）
- 功能/作用
- 新篇设计
- 替换确认

### 4.3 操作方法

1. 通读原文，标记所有具体人物、地点、事件、道具、台词
2. 按六类分类整理
3. 为每个关节点设计新变量，确保功能等价、情绪等价、逻辑自洽
4. 使用 `variable_annotator.py` 辅助扫描遗漏
5. 输出换元清单

## 5. Step 4：新变量方案设计

### 5.1 目标

基于骨架卡和换元清单，设计完整的新篇内容方案。

### 5.2 输出

新变量方案（YAML/JSON），包含：
- 新主角设定（姓名、身份、外貌、性格、目标、秘密）
- 新场景设定（地点、时代、环境、氛围）
- 新情节框架（触发事件、冲突、转折、高潮、结局）
- 新道具/意象设计
- 新关系网络
- 新台词/关键对话设计
- 情绪曲线映射

### 5.3 设计原则

- 新变量与原文无 N-gram 重合
- 新变量之间逻辑自洽
- 新变量能支撑原文骨架
- 新变量能支撑原文情绪曲线

### 5.4 操作方法

1. 使用 `new_variable_designer.py` 基于骨架和换元清单生成初稿
2. 人工审核每个变量是否符合功能等价原则
3. 调整不合适的变量
4. 输出最终方案

## 6. Step 5：按骨架逐段仿写

### 6.1 目标

根据骨架卡、字数预算表和新变量方案，逐段生成新短篇。

### 6.2 生成策略

- **逐段生成**，每段独立控制字数
- **段间不直接引用原文**，只参考骨架
- **使用新变量**，不保留原文具体内容
- **注入人味化**，规避 AI 检测

### 6.3 Prompt 核心约束

```
请根据以下骨架卡和字数预算，生成第 X 段。

约束：
1. 段落功能：[开头悬念/人物出场/冲突升级等]
2. 字数：[Y] 字（±5%）
3. 情绪：[具体情绪]
4. 句长分布：[短/中/长比例]
5. 对话占比：[Z%]
6. 使用新变量：[列出该段需要用的新变量]
7. 不得出现原文人物名、地名、道具名、台词
8. 不得复述原文情节
9. 语言要有呼吸感：长短句交错、偶尔短句、口语化自然
10. 避免 AI 高频词：furthermore, delve, tapestry, in conclusion 等
```

### 6.4 操作方法

1. 使用 `short_generator.py` 按段落调用 LLM
2. 每段生成后检查字数
3. 超字数则立即压缩，少字数则扩写
4. 生成完全部段落后拼接成初稿

## 7. Step 6：字数对齐修正

### 7.1 目标

确保新篇总字数和各段字数与预算对齐。

### 7.2 修正流程

1. 统计新篇总字数
2. 对比每段实际字数与预算
3. 找出偏差 >5% 的段落
4. 使用扩写或压缩技巧修正
5. 重新统计，直到达标

### 7.3 工具

- `word_count_adjuster.py`：自动检测偏差并给出修正建议

## 8. Step 7：反检测 & 原创性验证

### 8.1 目标

确保新篇 100% 原创且能规避 AI 检测。

### 8.2 三关验证

#### 关 1：原创性验证

- N-gram 重合检测（3-gram、5-gram）
- 变量残留扫描（人物名、地名、道具名、台词）
- 连续相同字检测（8 字以上）
- 句子结构相似度检测

**目标**：N-gram 重合率 <5%，变量残留率 0%，最长连续相同字 <8。

#### 关 2：字数验证

- 总字数 ±5%
- 每段字数 ±5%
- 对话/叙述/描写比例 ±10%

#### 关 3：反检测验证

- 困惑度 >80
- 突发性 >0.7
- AI 特征词 0 命中
- 模式化标记 0 命中

### 8.3 未通过处理

| 未通过项 | 处理方式 |
|----------|----------|
| 变量残留 | 重新设计变量，替换残留内容 |
| N-gram 重合 | 重写重合段落，进行语义替换 |
| 字数偏差 | 扩写或压缩 |
| 困惑度低 | 增加不完美表达、局部模糊、口语化 |
| 突发性低 | 增加短句、长句交错，打破均匀节奏 |
| AI 特征词 | 删除或替换为自然表达 |
| 模式化标记 | 打破结构化表达，增加自然转折 |

## 9. 最终输出

一篇成功完成的短篇换元仿写，应包含以下文件：

1. **原文副本**（`.md`）
2. **字数预算表**（`.md`）
3. **骨架卡**（`.yaml`）
4. **换元清单**（`.md`）
5. **新变量方案**（`.yaml`）
6. **新短篇小说**（`.md`）
7. **原创性验证报告**（`.md`）
8. **反检测验证报告**（`.md`）

## 10. 工作流注意事项

- 每一步都要检查输出是否偏离目标
- 骨架卡中如出现原文具体内容，必须删除
- 换元清单必须覆盖所有具体元素
- 生成段落时严格控制字数，不要等最后统一调整
- 验证未通过时，优先修正骨架和变量，而不是简单改写句子
- 全程保留中间文件，便于追溯和迭代

## 11. 质量检查清单

- [ ] 新篇总字数与原文 ±5%
- [ ] 骨架卡只包含写法，无具体内容
- [ ] 换元清单覆盖所有人物、地点、事件、道具、台词
- [ ] 新变量与原文无 8 字以上重合
- [ ] 新篇不保留原文人物名、地名、道具名
- [ ] 新篇情节与原文不同
- [ ] 5-gram 重合率 <5%
- [ ] 变量残留率 0%
- [ ] 困惑度 >80
- [ ] 突发性 >0.7
- [ ] AI 特征词 0 命中
- [ ] 新篇可读性流畅，无违和感
