# 01-what-is-substitution：什么是短篇换元仿写

## 1. 一句话定义

> **短篇换元仿写** = 保留原文的叙事骨架（结构、节奏、视角、语言、情绪），把人物、情节、地点、道具、台词等全部变量替换为原创内容，生成一篇**字数相近、风格神似、内容全新**的短篇小说。

借用数学中的"换元法"：

- 原文可视为函数 `y = f(x)`，其中 `f` 是叙事骨架，`x` 是内容变量
- 新短篇就是 `y' = f(x')`，骨架 `f` 不变，变量 `x` 全部换成 `x'`
- 结果：写法相同，故事完全不同

## 2. 核心公式

```
新短篇 = 原文骨架（保留） + 全新变量（替换） + 人味化（反检测） + 字数对齐（±5%）
```

| 成分 | 来源 | 是否保留 | 目的 |
|------|------|----------|------|
| 原文骨架 | 用户输入的短篇小说 | 保留 | 保证风格、节奏、结构相似 |
| 全新变量 | AI 原创设计 | 替换 | 保证 100% 内容原创 |
| 人味化 | 生成时主动注入 | 新增 | 规避 AI 检测 |
| 字数对齐 | 段落级预算控制 | 新增 | 保证与原文字数相近 |

## 3. 三层模型

### 3.1 骨架层（Skeleton Layer）—— 保留借鉴

骨架层是原文中可以借鉴的"写法"，包括：

- **结构骨架**：开头、发展、转折、结尾的段落安排
- **节奏骨架**：句子长短、段落长度、紧张/舒缓交替
- **视角骨架**：第一人称 / 第三人称、有限视角 / 全知视角
- **语言骨架**：短句偏好、修辞密度、对话比例、标点习惯
- **情绪骨架**：情绪曲线、情感转折、留白方式

**原则**：只学习方法，不搬运内容。

### 3.2 变量层（Variable Layer）—— 必须全换

变量层是原文中的具体内容，必须全部替换：

- 人物姓名、外貌、职业、关系
- 地点、时代、环境
- 核心事件、冲突、转折
- 台词、独白、对话
- 关键道具、意象
- 具体情节、细节

**原则**：凡是能让读者联想到原文的具体元素，全部替换。

### 3.3 人味层（Humanity Layer）—— 主动注入

人味层是为了让文本更像人类写作，规避 AI 检测，包括：

- 长短句交错、偶尔出现短句
- 口语化表达、重复、半重复
- 情绪跳跃、不完美逻辑
- 省略、留白、个人化判断
- 局部模糊、不解释所有细节

**原则**：不要生成"完美"的 AI 文本，要生成有呼吸感的文本。

## 4. 换元仿写 vs 直接改写 vs 洗稿

| 方式 | 做法 | 原创性 | 反检测 | 合法性 |
|------|------|--------|--------|--------|
| **直接改写** | 替换部分同义词，保留原句结构 | 低 | 差 | 风险高 |
| **洗稿** | 大量保留原文情节，换词换句 | 低 | 中 | 风险高 |
| **换元仿写** | 保留骨架，替换全部变量，从零创作 | 高 | 强 | 在合法授权下可接受 |
| **独立创作** | 完全不看原文，自由发挥 | 最高 | 最强 | 无风险 |

**关键区别**：换元仿写不是"改"原文，而是"借鉴写法后重新写"。

## 5. 适用场景

1. 学习某位作者的写法，但不想抄袭
2. 借鉴某篇短篇的结构和节奏，写自己的新故事
3. 用 AI 生成与人类文风相近的短篇样本
4. 批量产出同一风格但内容完全不同的小说

## 6. 红线规则

- 不得复制原文连续 8 个字以上
- 不得保留原文人物名、地名、道具名
- 不得复述原文核心情节
- 不得简单同义词替换原文句子
- 输出必须附带原创性验证报告

## 7. 成功标准

- 新短篇字数 = 原文 ±5%
- 5-gram 重合率 < 5%
- 变量残留率 = 0%
- 困惑度 > 80
- 突发性 > 0.7
- AI 特征词命中 = 0
- 骨架还原度（人工判断）> 80%
- 可读性流畅，无违和感
