# 05-short-verification：短篇三关验证流程

## 1. 验证总览

短篇换元仿写完成后，必须通过三关验证：

1. **原创性关**：确保内容 100% 原创，无原文残留
2. **字数关**：确保新篇字数与原文相近
3. **反检测关**：确保文本能规避 AI 检测

三关全部通过，方可输出终稿。

## 2. 第一关：原创性验证

### 2.1 验证项目

| 项目 | 目标 | 工具 |
|------|------|------|
| N-gram 重合率 | 5-gram < 5% | `originality_checker.py` |
| 变量残留率 | 0% | `originality_checker.py` |
| 连续相同字 | 最长 < 8 字 | `originality_checker.py` |
| 句子结构相似度 | < 30% | 自定义规则 |
| 人工复核 | 通过 | `originality-check/03-human-review.md` |

### 2.2 未通过处理

| 问题 | 处理策略 |
|------|----------|
| 5-gram 重合 > 5% | 重写重合段落，进行语义替换 |
| 变量残留 | 替换残留的人物名、地名、道具名、台词 |
| 连续相同字 > 8 字 | 改写该句，调整句式或替换关键词 |
| 句子结构相似 > 30% | 调整句式结构，增加或减少修饰 |

## 3. 第二关：字数验证

### 3.1 验证项目

| 项目 | 目标 | 工具 |
|------|------|------|
| 总字数 | 原文 ± 5% | `word_count_analyzer.py` |
| 每段字数 | 预算 ± 5% | `word_count_analyzer.py` |
| 句长分布 | 与原文相似 ± 10% | `word_count_analyzer.py` |
| 对话占比 | 与原文相似 ± 10% | `word_count_analyzer.py` |
| 叙述/描写占比 | 与原文相似 ± 10% | `word_count_analyzer.py` |

### 3.2 未通过处理

| 问题 | 处理策略 |
|------|----------|
| 总字数偏多 | 删除冗余形容词、合并句子、压缩对话 |
| 总字数偏少 | 增加细节、拆分句子、增加对话 |
| 某段字数偏差 | 单独扩写或压缩该段 |
| 对话占比偏差 | 调整对话长度或增减对话场景 |

## 4. 第三关：反检测验证

### 4.1 验证项目

| 项目 | 目标 | 工具 |
|------|------|------|
| 困惑度 | > 80 | `anti_detection_checker.py` |
| 突发性 | > 0.7 | `anti_detection_checker.py` |
| AI 特征词 | 0 命中 | `anti_detection_checker.py` |
| 模式化标记 | 0 命中 | `anti_detection_checker.py` |
| 外部检测工具 | AI 置信率 < 30% | 第三方工具 |

### 4.2 未通过处理

| 问题 | 处理策略 |
|------|----------|
| 困惑度低 | 增加不完美表达、局部模糊、口语化 |
| 突发性低 | 增加短句、长句交错，打破均匀节奏 |
| AI 特征词命中 | 删除或替换为自然表达 |
| 模式化标记命中 | 打破结构，增加自然转折 |
| 外部检测工具未通过 | 综合使用上述策略，迭代优化 |

## 5. 三关验证流程图

```
生成新短篇初稿
    ↓
[关 1] 原创性验证
    ↓
    ├─ 通过 → 进入关 2
    └─ 未通过 → 修正原创性问题 → 重新验证
    ↓
[关 2] 字数验证
    ↓
    ├─ 通过 → 进入关 3
    └─ 未通过 → 修正字数问题 → 重新验证
    ↓
[关 3] 反检测验证
    ↓
    ├─ 通过 → 输出终稿
    └─ 未通过 → 人味化修正 → 重新验证（从关 1 开始）
```

## 7. 验证迭代策略

### 7.1 单次未通过

- 关 1 未通过：重点检查换元清单，修正变量
- 关 2 未通过：使用字数修正工具，扩写或压缩
- 关 3 未通过：进入人味化流程，增加节奏扰动

### 7.2 多次未通过

如果迭代 3 次仍未通过，需要回到更早步骤：
- 反检测始终不过 → 检查骨架卡是否过于依赖原文结构，调整结构
- 原创性始终不过 → 检查新变量设计是否仍与原文有隐性相似，重新设计变量
- 字数始终对不齐 → 检查骨架卡段落功能是否合理，重新拆分段落

## 8. 人工复核

自动化验证后，必须进行人工复核：

- 通读新篇，判断可读性
- 检查是否有明显 AI 痕迹
- 检查是否有未替换的原文元素
- 检查情绪曲线是否自然
- 检查结尾是否自然，无刻意升华
- 检查对话是否符合人物身份
