# 03-human-review：人工复核清单

## 1. 人工复核的必要性

自动化工具可以检测 N-gram 重合、变量残留、困惑度、突发性等量化指标，但无法完全判断：
- 新篇是否真正"像原文"
- 新篇是否流畅可读
- 新篇是否有隐性的风格模仿过度
- 新篇是否保留了原文的"神韵"

因此，自动化验证之后，必须进行人工复核。

## 2. 复核维度

### 2.1 原创性复核

| 检查项 | 通过标准 |
|--------|----------|
| 新篇情节与原文不同 | 不能是简单的人物/地点替换版 |
| 新篇人物与原文不同 | 姓名、身份、外貌、关系全部原创 |
| 新篇结局与原文不同 | 结局导向、情感落点不同 |
| 没有复述原文核心事件 | 不能是原文的改写版 |
| 没有保留原文的特定意象 | 如原文是"雨"，新篇不能只是"雪"+ 其他完全相同 |

### 2.2 风格相似性复核

| 检查项 | 通过标准 |
|--------|----------|
| 新篇结构与原文相似 | 起承转合逻辑一致 |
| 新篇节奏与原文相似 | 紧张/舒缓段落位置一致 |
| 新篇视角与原文一致 | 人称、聚焦、范围一致 |
| 新篇语言风格与原文相似 | 句式、修辞密度、对话比例相似 |
| 新篇情绪曲线与原文相似 | 情绪变化趋势一致 |

### 2.3 可读性复核

| 检查项 | 通过标准 |
|--------|----------|
| 新篇读起来流畅 | 没有生硬的句子或段落 |
| 对话自然 | 符合人物身份和情境 |
| 逻辑自洽 | 人物动机和事件发展合理 |
| 没有过度的人味化痕迹 | 不要为了反检测而强行口语化 |
| 结尾自然 | 不突兀，不刻意升华 |

### 2.4 反检测复核

| 检查项 | 通过标准 |
|--------|----------|
| 没有 AI 高频词 | 不出现 furthermore、综上所述等 |
| 没有模式化结构 | 不机械地起承转合 |
| 有节奏变化 | 长短句交错 |
| 有不完美表达 | 有口语化、重复、省略等 |
| 没有过度总结 | 结尾不刻意说教 |

## 3. 复核流程

```
自动化验证三关通过
    ↓
人工通读新篇
    ↓
按四个维度打分
    ↓
记录问题
    ↓
决定是否修改
    ↓
如需修改 → 返回相应步骤
    ↓
无需修改 → 输出终稿
```

## 5. 常见人工复核问题

### 5.1 新篇过于"像原文"但工具检测通过

原因：新篇在骨架、情绪、节奏上极度模仿原文，但具体变量已替换。工具可能无法识别这种"隐性相似"。

处理：
- 调整新篇的某些结构节点
- 改变情绪转折方式
- 改变结尾导向

### 5.2 人味化过度

原因：为了反检测而加入大量口语化、重复、跳跃，导致文本不自然。

处理：
- 减少不必要的口语化
- 保持情绪的内在逻辑
- 删除为反检测而添加的冗余表达

### 5.3 风格相似度不足

原因：换元过于彻底，导致新篇失去了原文的写法特征。

处理：
- 重新检查骨架卡
- 在句式、修辞密度、对话比例上向原文靠拢
- 调整段落长度分布

### 5.4 字数对齐但节奏不对

原因：为了凑字数，在某些段落添加了无关内容或过度压缩。

处理：
- 重新分配段落字数
- 在保证节奏的前提下调整字数

## 6. 复核红线

以下情况必须重新创作：
- 新篇情节明显是原文的改写版
- 新篇保留了原文人物名或核心道具名
- 新篇有 8 字以上连续相同表达
- 新篇有 AI 检测工具高置信率判定为 AI 生成
- 新篇字数严重偏离原文（> ±10%）
- 新篇可读性差，有明显 AI 痕迹
