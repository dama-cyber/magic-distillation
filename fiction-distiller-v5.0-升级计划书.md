| 问题 | 修复 | 验证结果 |

### 1.2 仍未突破的核心瓶颈

**AIGC 值 0.8142 → 目标 <0.5**，差距 0.31。根因不是某几个词或某几个句子的技术问题，而是**生成范式的结构性缺陷**：

```
v4.1 生成范式：
原文骨架（事件顺序 A→B→C→D）
    ↓
换元（人名/地名/道具替换）
    ↓
逐段仿写（保持段落功能、情绪、字数）
    ↓
输出：骨架相同 + 表面不同 = 统计特征仍像 AI
```

检测器（DetectGPT / ZeroGPT / GPTZero）识别的不是"内容是否抄袭"，而是**文本在概率空间中的位置**。AI 生成的文本位于语言模型的 log-probability 局部最大值（负曲率区域）——微小改写后概率必然下降。人类文本则分布在曲率接近零的区域。

**v4.1 的问题**：逐段仿写时，LLM 仍被骨架约束在"最合理的下一句话"的生成路径上，产出的文本在概率空间中仍靠近 AI 的局部峰值。

### 1.3 需要升级的认知框架

| 维度 | v4.1 思维 | v5.0 思维 |
|------|----------|----------|
| 原创性 | 内容不重复 = 原创 | **统计分布不重复** = 原创 |
| 人类感 | 避免 AI 词、增加短句 | **主动制造概率层面的混乱** |
| 骨架使用 | 严格遵循原文骨架 | **从骨架中提取"写法规则"，再按新内容重构骨架** |
| 节奏控制 | 保持与原文相近 | **有意打破原文节奏，制造人类特有的"不规律"** |

## 二、v5.0 核心设计理念

### 2.1 双轨原创性

```
原创性 ≠ 内容不重复
原创性 = 内容不重复 + 统计分布不重复 + 生成路径不可追溯
```

- **内容层**：人物、地点、情节、台词全部替换（v4.1 已实现）
- **统计层**：句长分布、段落长度、信息密度、连接词频率与原文不同（v5.0 新增）
- **路径层**：生成时的 token 选择路径不落在原文的概率峰值上（v5.0 核心）

### 2.2 "有规律的混乱"——人类写作的本质

人类写作的特征不是"完美"，而是**有节奏的混乱**：

- 有节奏：情绪起伏、张弛有度
- 混乱：句子长度不均、信息密度波动、偶尔离题、语法不完美

AI 写作的特征是**无规律的整齐**：
- 每段长度相近、每句复杂度相近、过渡永远标准、语法永远正确

**v5.0 的核心任务**：在保持"有节奏"的前提下，系统性地注入"混乱"。

### 2.3 三层人类化改造

```
┌─────────────────────────────────────────────┐
│  第一层：骨架级改造（影响最大，决定 0.5→0.3）  │
│  改变事件顺序、信息密度分布、段落功能分配      │
├─────────────────────────────────────────────┤
│  第二层：语言级改造（影响中等，决定 0.7→0.5）  │
│  极端突发性、口语化、不完美语法、碎片化        │
├─────────────────────────────────────────────┤
│  第三层：内容级改造（影响持久，决定 0.3→0.1）  │
│  个人叙事印记、文化特异性、主观判断            │
└─────────────────────────────────────────────┘
```

## 三、具体升级模块

### 模块一：骨架提取升级（skeleton_extractor.py → v2.0）

#### 3.1.1 当前问题

v4.1 的骨架卡提取的是原文的**段落功能、情绪、字数**，本质上是复制了原文的骨架结构。新篇按此骨架生成，自然与原文在统计分布上高度相似。

#### 3.1.2 升级方案

骨架卡从"复制结构"升级为**"提取写法规则 + 允许结构变异"**：

```yaml
# v4.1 骨架卡（复制型）
段落1:
  功能: 开头悬念
  字数: 927
  情绪: 紧张
  句长分布: 短30% 中50% 长20%

# v5.0 骨架卡（规则型 + 变异许可）
写法规则:
  - 开头用环境描写制造压抑感（不指定具体环境）
  - 前3段内必须出现主角的某个身体缺陷或习惯动作
  - 对话占比不超过30%
  - 至少有一段纯心理独白，不推进情节

结构变异许可:
  - 允许在事件A和B之间插入无关的回忆（500字以内）
  - 允许某个事件不写过程，只写结果
  - 允许结尾不总结，戛然而止

节奏要求:
  - 段落长度方差必须 > 50%（即最长段至少是最短段的3倍）
  - 必须包含至少3个单句段（<=15字）
  - 必须包含至少1个超长段（>1500字）
```

#### 3.1.3 实施

1. 重写 `skeleton_extractor.py`，输出"规则型骨架卡"
2. 在骨架卡中增加"结构变异许可"字段
3. 新增 `rhythm_variance` 字段，强制要求段落长度方差

### 模块二：生成策略升级（short_generator.py → v2.0）

#### 3.2.1 当前问题

v4.1 的生成 prompt 约束了段落功能、字数、情绪、句长分布。这些约束过于精确，导致 LLM 在生成时走在"最合理"的路径上。

#### 3.2.2 升级方案

**策略 A：概率扰动生成（核心创新）**

不直接生成最终文本，而是生成 3 个版本后选择性拼接：

```python
# 伪代码
versions = []
for temp in [0.7, 1.0, 1.3]:
    versions.append(generate(paragraph_prompt, temperature=temp))

# 拼接策略：
# - 开头选 temp=0.7 版本（连贯）
# - 中间选 temp=1.3 版本（混乱）
# - 结尾选 temp=1.0 版本（平衡）
# - 在过渡处故意制造语义跳跃
final_text = stitch_with_jumps(versions)
```

**策略 B：强制节奏破坏**

在 prompt 中增加"反均匀"指令：

```
生成约束：
1. 本段字数：[Y] 字（±20%，不再是±5%）
2. 句长分布：必须包含至少1个3字以下短句和1个50字以上长句
3. 段落节奏：如果上段是长段，本段必须是短段（<=200字）
4. 信息密度：允许有1-2句与主线无关的闲笔（环境、回忆、走神）
5. 连接词：禁止使用"然而""此外""因此""与此同时"——用口语化替代
6. 对话：必须包含至少1处打断、1处答非所问、1处沉默（用动作代替回答）
```

**策略 C：人称与视角跳切**

在适当位置允许视角微调：
- 第三人称中突然插入叙述者主观判断（"说来也怪""那年头就这样"）
- 在紧张场景中突然拉开距离，描写无关的环境细节

1. 重写 `short_generator.py`，支持多温度生成 + 拼接
2. 更新 `p04-paragraph-generate.md`，加入"反均匀"约束

### 模块三：后处理人类化（新增 humanize.py）

#### 3.3.1 定位

在生成初稿后、验证前，增加一个强制的人类化后处理步骤。这一步不依赖 LLM，而是用规则 + 随机扰动主动制造"混乱"。

#### 3.3.2 处理规则

```python
def humanize(text, target_burstiness=0.85):
    """人类化后处理"""
    
    # 1. 强制插入极端短句（每1000字3-5个）
    text = inject_micro_sentences(text, density=0.004)
    
    # 2. 强制段落长度方差（选择性地合并或拆分段落）
    text = adjust_paragraph_variance(text, min_ratio=3.0)
    
    # 3. 连接词替换（标准→口语化）
    text = replace_connectors(text)
    
    # 4. 对话碎片化
    text = fragment_dialogue(text)
    
    # 5. 插入闲笔（每2000字1处，与主线无关的感官细节）
    text = insert_digressions(text, density=0.0005)
    
    # 6. 制造"人类错误"（轻微的语法不完美）
    text = inject_minor_imperfections(text, rate=0.002)
    
    return text
```

#### 3.3.3 各子功能详解

**inject_micro_sentences**：在段落内部或段间插入 1-3 字短句
```
原文：她走进房间，发现桌上有一封信。
处理后：她走进房间。桌上有一封信。就这么摆着。
```

**adjust_paragraph_variance**：如果所有段落长度相近，强制拆分最短段、合并最长段

**replace_connectors**：
| 原文 | 替换 |
|------|------|
| 然而 | 可、不过、话说回来、倒也是 |
| 此外 | 还有、再说、顺便提一句 |
| 因此 | 于是、所以、这下、到头来 |
| 与此同时 | 那会儿、正当这时、偏偏这时候 |
| 最后 | 临了、末了、到头来、到头来还是 |

**fragment_dialogue**：
- 把完整回答改成打断（"你别——"）
- 加入不回应（用动作代替回答）
- 加入重复和口吃（"我、我……"）

**insert_digressions**：在叙事中插入与主线无关但真实的细节
```
（在紧张情节中突然插入）
窗外的梧桐树那年刚栽，叶子还稀稀拉拉的，
她记得小时候外婆家也有这么一棵。
```

**inject_minor_imperfections**：
- 一处 run-on sentence（逗号拼接）
- 一处 fragment（不完整句）
- 一处口语化量词错误（如果符合角色）
- 一处突然的省略（"反正就是……那样吧"）

#### 3.3.4 实施

1. 新建 `humanize.py` 工具
2. 在七步流程中，Step 5 和 Step 6 之间插入 Step 5.5：人类化后处理
3. 所有规则可配置（密度、强度）

### 模块四：反检测验证升级（anti_detection_checker.py → v3.0）

#### 3.4.1 当前问题

v4.1 检测了 Burstiness、短句占比、AI 特征词、模式化标记。但未检测：
- 段落长度方差（段落过于均匀是强 AI 信号）
- 连接词多样性（重复使用标准连接词）
- 信息密度波动（每段字数过于接近预算）
- 对话自然度（对话是否过于"整洁"）

#### 3.4.2 新增检测项

| 新增指标 | 人类基准 | AI 特征 | v5.0 目标 |
|---------|---------|---------|----------|
| **段落长度方差比** | 最长/最短 ≥ 3x | 最长/ shortest ≤ 1.5x | ≥ 3x |
| **连接词多样性** | 同义替换率 ≥ 60% | 重复使用标准词 | ≥ 60% |
| **信息密度波动** | 相邻段落字数差 ≥ 50% | 相邻段落字数差 ≤ 20% | ≥ 50% |
| **对话碎片化率** | ≥ 30% 对话含打断/省略 | ≤ 10% | ≥ 30% |
| **闲笔密度** | 每 2000 字 1 处 | 0 | ≥ 0.0005 |
| **视角跳切** | 至少 1 处 | 0 | ≥ 1 |

### 模块五：原创性验证升级（originality_checker.py → v2.0）

#### 3.5.1 当前问题

v4.1 检测了 N-gram 重合、变量残留、LCS、结构相似度。但未检测：
- **骨架相似度**：事件顺序、段落功能分布是否过于相似
- **情绪曲线相似度**：情绪起伏的节奏是否与原文一致
- **信息密度分布相似度**：每段字数占比是否与原文一致

#### 3.5.2 新增检测项

```python
# 骨架相似度检测
def check_skeleton_similarity(original_skeleton, new_text):
    # 提取新篇的段落功能分布
    new_functions = extract_paragraph_functions(new_text)
    # 对比原文骨架的功能序列
    similarity = sequence_similarity(original_skeleton['functions'], new_functions)
    return similarity < 0.5  # 功能序列相似度不超过50%

# 情绪曲线相似度
def check_emotion_curve_similarity(original_curve, new_text):
    new_curve = extract_emotion_curve(new_text)
    # 计算两条曲线的动态时间规整距离
    dtw_distance = dtw(original_curve, new_curve)
    return dtw_distance > threshold

# 信息密度分布相似度
def check_density_distribution_similarity(original_budget, new_text):
    original_distribution = [p['target_chars'] for p in original_budget['paragraphs']]
    new_distribution = [count_chinese_chars(p) for p in split_paragraphs(new_text)]
    # 计算分布的相关系数
    corr = correlation(original_distribution, new_distribution)
    return corr < 0.6  # 分布相关性不超过0.6
```

```python
humanize.py
人类化后处理工具 v1.0

"""

import random

CONNECTOR_MAP = {

    '因此': ['于是', '所以', '这下', '到头来'],
    '与此同时': ['那会儿', '正当这时', '偏偏这时候'],
    '最后': ['临了', '末了', '到头来'],
}

# 碎片化对话模板
DIALOGUE_FRAGMENTS = [
    '（打断）"你别——"',
    '（沉默）他张了张嘴，没出声。',
    '（重复）"我、我……"',
    '（答非所问）"你吃了吗？""天挺热的。"',
]

# 闲笔模板（需根据具体场景填充）
DIGRESSION_TEMPLATES = [
    '窗外的{植物}那年刚栽，{状态}，她记得{回忆}。',
    '空气里有{气味}的味道，{时间}的{地点}总是这样。',
    '{物品}上落了灰，{细节}，谁也没注意到。',
]

def humanize(text: str, config: dict = None) -> str:
    """主入口"""
    cfg = config or CONFIG
    
    text = inject_micro_sentences(text, cfg['micro_sentence_density'])
    text = adjust_paragraph_variance(text, cfg['paragraph_variance_ratio'])
    text = replace_connectors(text)
    text = fragment_dialogue(text, cfg['dialogue_fragmentation_rate'])
    text = insert_digressions(text, cfg['digression_density'])
    text = inject_minor_imperfections(text, cfg['imperfection_rate'])
    
    return text
```

## 九、结语

v4.1 解决了"能不能跑通"的问题。v5.0 要解决的是"像不像人类"的问题。

这不是简单的参数调优或 prompt 修补，而是**生成范式的重构**——从"遵循骨架的仿写"转向"提取规则后的自由创作"，从"避免 AI 特征"转向"主动注入人类特征"。

**核心信念**：人类写作的价值不在于完美，而在于有选择的混乱。v5.0 的目标不是让 AI 写出"更好的"小说，而是让 AI 写出"更像人写的"小说——有呼吸、有瑕疵、有意外、有灵魂。
